## Setup and Installation

### Prerequisites

Before using AnyHtmlOllama, make sure you have:

1. **Ollama installed and running**
   - Download from [ollama.ai](https://ollama.ai)
   - Install at least one model (gemma:2b, llama3:8b, or similar)
   - Ensure Ollama is running in the background

2. **Python for the local server**
   - Required to run the application in your browser

### Starting the Application

1. **Windows users**:
   - Double-click the `start_server.bat` file
   - A command prompt window will open and start the server
   - Your default browser should open automatically to the application

2. **macOS/Linux users**:
   - Open a terminal in the AnyHtmlOllama directory
   - Make the script executable: `chmod +x start_server.sh`
   - Run the script: `./start_server.sh`
   - Open your browser to http://localhost:8000/AnyHtmlOllama.html

## Interface Overview

The application is divided into two main panels:

### Left Panel: Input & Configuration

- **Model Selection**: Choose which Ollama model to use
- **Prompt Input**: Enter your description for the tool you want to create
- **Saved Tools**: Browse and load previously created tools
- **Saved Prompts**: Browse and load previously used prompts

### Right Panel: Preview & Output

- **Code Preview**: View the generated HTML/JavaScript code
- **Live Preview**: See your tool in action
- **Tool Name**: Enter a name for saving your tool
- **Action Buttons**: Transform, save, or copy your tool

## Generating a Tool

1. **Select a model** from the dropdown (make sure Ollama is running)
2. **Enter a prompt** describing the tool you want to create
   - Be specific about functionality, design, and features
   - See [Best Practices for Prompts](#best-practices-for-prompts) below
3. **Click "Generate Tool"**
4. Wait for the AI to create your tool (usually takes 5-30 seconds)
5. The generated code will appear in the Code Preview panel

## Previewing Your Tool

There are two ways to preview your generated tool:

1. **Live Preview Tab**:
   - Click the "Live" button in the right panel
   - See your tool rendered in an iframe
   - This is sandboxed for security

2. **Transform Button**:
   - For a full-page experience, click "Transform Now"
   - Your entire browser tab becomes the tool
   - Use the "Return to AnyHtmlOllama" button to go back

## Saving Tools

To save a generated tool:

1. **Enter a name** in the "Tool Name" field
2. **Click "Save Tool"**
3. Choose where to save:
   - Ideally, select the `saved_tools` directory in your AnyHtmlOllama folder
   - If using the app without a local server, it will download to your Downloads folder

The tool will be saved as:
- `[tool_name].html` - The actual tool code
- `[tool_name].md` - A markdown file containing the original prompt

## Loading and Modifying Existing Tools

To load a previously saved tool:

1. **Go to the "Saved Tools" tab** in the left panel
2. **Browse the list** of saved tools (click "Refresh" if needed)
3. **Click on a tool name** to select it
4. **Click "Load Tool"** to load it into the editor

To modify a tool:
1. Load the tool
2. Review the code in the Code Preview
3. If you want to regenerate with modifications:
   - Go to the "Saved Prompts" tab
   - Find and load the corresponding prompt
   - Modify the prompt and regenerate

## Best Practices for Prompts

For the best results, make your prompts:

1. **Specific**: Clearly describe functionality, design elements, and behaviors
2. **Structured**: Break down complex requirements into logical sections
3. **Focused**: Request one main feature or tool rather than multiple unrelated features
4. **Technical when needed**: Include specific technical requirements if you need them

### Example Prompts

Good examples:

- "Create a to-do list app with the ability to add, check off, and delete items. Use a clean, modern interface with soft colors."
- "Make a random password generator that can create passwords of varying length and complexity, with options for including numbers, symbols, and uppercase letters."
- "Build an interactive color picker that shows RGB and HEX values, with a preview box that updates in real-time."

## Troubleshooting

### Common Issues and Solutions

**Problem**: Models don't appear in the dropdown
- **Solution**: Ensure Ollama is running (check http://localhost:11434 in your browser)
- **Solution**: Make sure you have at least one model installed (`ollama list`)

**Problem**: Tool generation fails or times out
- **Solution**: Try a smaller/faster model like gemma:2b
- **Solution**: Simplify your prompt and try again
- **Solution**: Check Ollama's resource usage; it might be overloaded

**Problem**: Can't save files to the saved_tools directory
- **Solution**: Run the application using the provided server scripts
- **Solution**: Check file permissions in your AnyHtmlOllama directory

**Problem**: Tools don't appear in the Saved Tools list
- **Solution**: Click the "Refresh" button
- **Solution**: Make sure you're running the app through the local server
